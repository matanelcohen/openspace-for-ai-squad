In ingestion-status.tsx, handleIngest creates two competing setTimeouts (5s and 120s) that both clear the same interval and call setIngesting(false). The 5s timeout fires first, making the 120s timeout dead code that redundantly clears an already-cleared interval. Worse, if the component unmounts before timers fire, all timers leak and call refetch/setState on an unmounted component. Fix: consolidate to a single polling strategy with proper useEffect cleanup, or ideally use a WebSocket signal for ingestion completion instead of hardcoded timers.
