handleIngest() creates a setInterval (3s poll) and two competing setTimeouts (5s and 120s) — both calling setIngesting(false) — with no cleanup on unmount. If the user navigates away mid-ingestion, all timers continue firing setState on an unmounted component. The dual setTimeout also creates a race where ingestion appears done after 5s but the 120s timer re-sets state later. Fix: track timers in a ref, clear all on unmount, and consolidate to a single timeout with an isMounted guard.
