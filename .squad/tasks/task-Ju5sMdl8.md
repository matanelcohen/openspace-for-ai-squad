Several components fire async operations (fetch/setTimeout) without cleanup on unmount: (1) use-voice-session.ts POSTs to /api/voice/speak with no AbortController — if user closes the session, the pending call still resolves and mutates state. (2) workflows/compose/page.tsx uses setTimeout(router.push, 1000) in onSuccess — if user navigates away before 1s, the stale push fires. Both cause 'state update on unmounted component' errors. Fix: wrap fetches with AbortController returned from useEffect cleanup; move the redirect timeout into a useEffect gated on a 'saved' flag so cleanup cancels it.
