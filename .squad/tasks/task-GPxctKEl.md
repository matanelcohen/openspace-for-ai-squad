**Result:**
Done. Lint + prettier passed, committed.

**Root cause:** The `ws.onerror` handler in `terminal.tsx` was empty, assuming `onclose` always fires after `onerror`. The WebSocket spec doesn't guarantee this — when a connection error occurs (server down, network drop, DNS failure), `onerror` can fire without `onclose`, so `scheduleReconnect()` never gets called. Terminal stays stuck in "Connecting…" forever.

**Fix:** Added `ws.close()` in `onerror` to ensure `onclose` fires and triggers reconnection — same pattern already used in `use-websocket.ts`.
