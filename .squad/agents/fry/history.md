## Learnings

<!-- Append new learnings below. Each entry is something lasting about the project. -->

- **2026-03-23 — PRD published at `docs/prd.md`.** Read this before starting any feature work. It defines v1 scope: web-only, single-squad, 6 core features (Dashboard, Task Board, Activity Feed, Voice, Chat, Real-time Monitoring). Frontend is Next.js + React. See open decisions in §8 of PRD.
- **2026-03-23 — P0-2 Frontend scaffolding complete.** Next.js 14 App Router with Tailwind CSS (dark/light via CSS vars + `class` strategy), shadcn/ui (New York style, Button + Card components), TanStack Query provider, Inter font, sidebar layout with nav to 5 routes: `/` (Dashboard), `/tasks`, `/chat`, `/decisions`, `/voice`. Vitest + React Testing Library configured with 13 passing tests. `next.config.ts` renamed to `.mjs` — Next.js 14.2 doesn't support TS config files. Explicit `types` field added to tsconfig to avoid root-level `@eslint/js` type pollution.
- **2026-03-23  Phase 0 foundation is complete.** The repo now has the monorepo baseline, Next.js frontend shell, shared `@openspace/shared` contracts, CI workflow, and repo-wide quality tooling. Frontend Phase 1 work can now focus on real data flows and `.squad/`-backed product screens instead of scaffolding.
- **2026-03-23 — P1-4 through P1-7 REST API endpoints complete.** Created 4 Fastify route plugins in `apps/api/src/routes/`: `agents.ts` (GET list + GET detail), `tasks.ts` (full CRUD with filters, status/priority patch), `decisions.ts` (list, detail, full-text search), `squad.ts` (composite overview). All routes registered in `app.ts` via Fastify plugin pattern with `/api` prefix. `SquadParser` extended with `getTasks()` and `getTasksDir()` to integrate task data into the overview. `buildApp()` now accepts `squadDir` option for testability. 57 route tests covering happy paths, 400 validation, 404 not-found, filter combinations, and search. Routes use `@openspace/shared` types and the existing `TaskWriter` CRUD layer. Key patterns: Fastify type augmentation via `declare module 'fastify'` for `squadParser` decoration; search route registered before parameterised `:id` route to avoid capture conflicts.
- **2026-03-23 — P2-1 through P2-5 complete: UI component library, app shell, dashboard, and Kanban task board.** Full shadcn/ui component library installed: Badge, Avatar, Skeleton, Input, Textarea, Tabs, Select, Table, Dialog, DropdownMenu. Three custom components built: StatusBadge (color-coded agent status), PriorityBadge (P0-P3 with color), AgentAvatar (emoji/initials with agent-specific colors). App shell enhanced with TopBar showing squad name and dark mode toggle. API client utility (`src/lib/api-client.ts`) wraps fetch with typed error handling and JSON defaults. TanStack Query hooks (`use-agents`, `use-squad`, `use-tasks`) with 30s auto-refresh and optimistic updates on task status mutation. Dashboard page renders summary stats (animated number transitions) from `/api/squad` and agent grid from `/api/agents` with full skeleton loading and error states. Task board Kanban view with 5 columns (Backlog, In Progress, In Review, Done, Blocked) using `@dnd-kit/core` + `@dnd-kit/sortable` for drag-and-drop between columns. On drop, fires `PATCH /api/tasks/:id/status` with optimistic UI. `next.config.mjs` updated with `resolve.extensionAlias` to handle `@openspace/shared` ESM `.js` → `.ts` resolution. 83 tests passing across 11 test files. Production build clean.
